# 演示素材方案

如果你想让这个仓库更容易被点开、读完、再顺手点 star，演示素材的价值通常比多写几段介绍更高。

这份方案不是让你一次拍很多内容，而是优先做最值得的那几个素材。

## 优先级最高的素材

### 1. 首页 GIF

建议时长：20 到 40 秒

建议内容：

1. 打开一个“普通但不够吸引人”的仓库首页
2. 调用 `github-launch-kit`
3. 展示它输出的 README 改造建议
4. 切到改造后的首页结构

这类 GIF 的作用不是完整教学，而是让用户 5 秒内理解：

- 它处理什么问题
- 它的输出长什么样
- 为什么值得继续看

### 2. Before / After 对比图

建议做一张双栏图：

- 左边：改造前 README 顶部
- 右边：改造后 README 顶部

图里重点突出三类变化：

- 一句话定位更清楚
- 证据更靠前
- 快速开始更短

这张图可以同时放在 README、X 帖子、Reddit 帖子和封面图里复用。

### 3. 一个真实案例截图

如果你后面拿它改过自己的其他仓库，最好补一张真实案例截图：

- 原仓库是什么类型
- 改了哪些首页模块
- 改造后有什么直观提升

不必夸大结果，只要真实，可信度就会比“功能列表”高很多。

## 推荐素材结构

如果你要一次性做完首发素材，建议按这个顺序准备：

1. 封面图：一句话价值 + 项目名
2. GIF：展示工作流
3. Before / After：展示效果
4. 一张真实案例截图：展示可信度

## 录制建议

- 保持画面干净，不要录太多无关窗口
- 尽量控制在一个完整流程里，不要来回切太多次
- 展示结果比展示输入更重要
- 如果终端输出很长，只截最有代表性的部分

## README 里怎么放

推荐顺序：

1. 顶部预览图
2. 一句话价值说明
3. GIF 或案例图
4. 关键能力列表
5. 安装和使用方式

## 最小可执行版本

如果你现在没时间做完整素材，先做下面两个就够了：

1. 一张 Before / After 对比图
2. 一个 20 秒以内的短 GIF

这两个素材是最容易显著提升首页吸引力的。
