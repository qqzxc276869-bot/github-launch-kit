## Hacker News summary

I kept seeing technically solid repos get ignored because the GitHub presentation was weak: vague README openings, no proof near the top, long setup, and no clear reason to care in the first 10 seconds.

So I made a Codex skill that reviews a repository like a launch surface. It helps tighten the one-line pitch, restructure the README, improve quickstart clarity, suggest proof sections, and draft launch copy without inventing hype.

The goal is not fake growth. The goal is making useful work easier to discover and trust.
